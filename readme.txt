-L’applicazione è il progetto 2 Memorandum, sviluppato per iOS

-L’applicazione sfrutta il pattern architetturale Model-view-controller, dove le classi nella cartella “Model” si occupano di gestire i dati dell’applicazione, i quali vengono poi passati ai vari View controller (uno per ogni schermata dell’applicazione) che li gestiscono per poi passarli alle View vere e proprie definite nello storyboard, che hanno lo scopo di rappresentare i dati

-l’applicazione utilizza un navigation controller, per garantire un passaggio tra le varie schermate intuitivo, e rendere sempre disponibile la possibilità di premere il tasto indietro in alto a destra per tornare alla View principale

-l’applicazione è completamente responsive grazie alle constraint inserite per ogni elemento grafico, rendendo possibile l’utilizzo dell’app da qualunque dispositivo con qualunque grandezza di schermo

-L’applicazione infine utilizza delle Table View per mostrare i dati in maniera ordinata e intuitiva per l’utente
